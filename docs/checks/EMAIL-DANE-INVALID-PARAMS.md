# EMAIL-DANE-INVALID-PARAMS

**DANE/TLSA records have valid parameters**

## Description

TLSA records must have Usage (0–3), Selector (0–1), and Matching Type (0–2) within valid ranges per RFC 6698 §7.2. Invalid values cause DANE-aware MTAs to reject mail delivery.
