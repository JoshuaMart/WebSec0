A single deployable artefact. No external Redis, no database, no Chromium,
no Postfix. The Astro frontend is embedded via `//go:embed all:dist`.

## Components

### HTTP API (`internal/api`)

- **Spec-first** вҖ” `api/openapi.yaml` is the source of truth.
- Serveur and client Go code are generated by [ogen](https://github.com/ogen-go/ogen)
  into `pkg/client/` (the server reuses the generated types).
- Routing is `net/http` + `github.com/go-chi/chi/v5` for composable
  middlewares (`request-id`, `recover`, `slog-access-log`, `cors`).
- The OpenAPI document is embedded via `go:embed` and served on
  `/api/v1/openapi.json`. A Spectral lint job in CI blocks regressions.
- A `verify-codegen` workflow detects drift between the spec and the
  generated code and fails the PR.

### Scanner orchestrator (`internal/scanner`)

Three-level concurrency, applied uniformly:

1. **Process-wide** вҖ” `semaphore.NewWeighted(maxConcurrentScans)` (default 50).
2. **Per scan** вҖ” `errgroup.WithContext` + `g.SetLimit(10)`.
3. **Per check** вҖ” `context.WithTimeout(ctx, per_check_timeout)` (default 8 s).

A DNS cache (`map[string][]net.IP` protected by `sync.RWMutex`) is shared
for the duration of a scan to avoid redundant lookups. `errors.Join`
collects partial errors. There is **no** retry logic вҖ” a timeout is itself
information (`status: "error"` on the relevant finding), not something to
mask.

### Check registry (`internal/checks`)

Every check satisfies the `Check` interface (`ID`, `Family`,
`DefaultSeverity`, `Run`) and self-registers via `init()`. The catalog
endpoint (`GET /api/v1/checks`) reflects the runtime registry вҖ” there is
no static manifest to keep in sync.

Optional `Describer` interface enriches the catalog entry with a title,
long description, and RFC references (rendered into the per-check pages
under [`docs/checks/`](./checks/) by `scripts/gen-checks-docs.sh`).

### Storage (`internal/storage`)

```go
type ScanStore interface {
    Put(ctx context.Context, scan *Scan, ttl time.Duration) error
    Get(ctx context.Context, id string) (*Scan, error)
    Delete(ctx context.Context, id string) error
    UpdateStatus(ctx context.Context, id string, fn func(*Scan) error) error
}
```

Three implementations:

| Backend     | Use case                                                  |
|-------------|-----------------------------------------------------------|
| `memory`    | MVP / single instance вҖ” `patrickmn/go-cache`              |
| `ristretto` | High concurrency, TinyLFU, hard memory ceiling            |
| `redis`     | Multi-replica, horizontal scaling (V2 roadmap)            |

TTL defaults to 24 h. GC every 5 minutes. No on-disk persistence at MVP
(memory/ristretto). With Redis, AOF/RDB depend on the deployer's Redis
config.

### TLS stack вҖ” hybrid

| Layer                                     | Implementation                              |
|-------------------------------------------|---------------------------------------------|
| TLS 1.2 / 1.3, chain validation, ALPN, OCSP stapling | `crypto/tls` stdlib              |
| TLS 1.0 / 1.1, legacy cipher enumeration  | Raw socket probes (`internal/scanner/tls/probes/`) |
| **SSLv3 / SSLv2 (probe yes/no)**          | **Hand-crafted ClientHello** over raw TCP   |
| Heartbleed                                | Raw socket probe (Heartbeat extension)      |
| CT logs / SCT validation                  | `crypto/x509` + OID `1.3.6.1.4.1.11129.2.4.2` |

The raw probes do **not** depend on `zcrypto`. To detect that a server
*accepts* SSLv2/SSLv3, completing the handshake is unnecessary вҖ” observing
the first record is enough (`ServerHello` вҶ’ accepted, `Alert`/RST вҶ’ rejected).

### Frontend (`web/` вҶ’ `internal/webfs/dist/`)

- **Astro 6** in static mode, no SSR.
- **Tailwind CSS** + **Alpine.js** (15 kB) island on the live-progress page.
- Build output lands in `internal/webfs/dist/` and is `go:embed`-ed at
  compile time. Single binary, no Node.js at runtime.
- Routes: `/`, `/scan/{guid}`, `/about`, `/checks`, `/docs/api`,
  `/legal/{tos,privacy}`.

## Async request-reply

A full scan takes 30 s вҖ“ 2 min. Holding a synchronous HTTP connection is
an anti-pattern (proxy timeouts, poor agent UX). The pattern is the
[Asynchronous Request-Reply](https://learn.microsoft.com/en-us/azure/architecture/patterns/async-request-reply)
recipe (RFC 9110 friendly):

- `POST /api/v1/scans` вҶ’ **`202 Accepted`** with `Location:
  /api/v1/scans/{guid}` and `Retry-After: 5`.
- `GET /api/v1/scans/{guid}` вҶ’ **always `200 OK`** with a structured
  `status` field. No ambiguous `404`/`202` on polling.
- `?wait=30s` (or `options.wait_seconds` on POST) for short synchronous
  waits. If the scan exceeds `wait`, the API falls back to async.
- `GET /api/v1/scans/{guid}/events` is SSE (`text/event-stream`) with
  `Last-Event-ID` and native `EventSource` reconnection.

WebSocket was rejected: SSE is unidirectional (sufficient here), simpler
to proxy, and consumable via `curl -N` (an asset for AI agents).

## Privacy model

- **Zero-log domains by default** (`logging.log_targets: false`). Access
  logs include the GUIDv4, never the target hostname.
- **GUIDv4** report URLs (122 bits of entropy via `crypto/rand`) вҖ” not
  enumerable.
- **No public listing** вҖ” no endpoint enumerates existing scans.
- **Private mode** вҖ” `options.private: true` returns a 256-bit token
  required for `GET`/`DELETE`. Returned once.
- **Anti-SSRF** вҖ” RFC 1918, link-local, loopback, IPv6 ULA, CGNAT all
  refused on the public instance (configurable for intranet self-hosting).
- **IP anonymization** for rate-limit bookkeeping вҖ” last octet zeroed
  (IPv4) or `/64` (IPv6), retained 7 days max. Legal basis: GDPR
  art. 6(1)(f) legitimate interest.

## Configuration

Layered, highest priority first:

1. CLI flags
2. Environment variables prefixed `WEBSEC0_`
3. Config file (`/etc/websec0/config.yaml` or `--config`)
4. Built-in defaults (see [`internal/config/defaults.go`](../internal/config/defaults.go))

A reference `config.yaml` lives in [`docs/self-hosting.md`](./self-hosting.md#configuration).
